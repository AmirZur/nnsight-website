nnsight.module#

The Module object acts as the entrypoint into the interwoven intervention graph. Through it, one can manipulate the inputs and outputs of Modules as a model is running. Specifically, the .output and .input attributes are the root nodes of the intervention graph, and all operations performed on these Proxy objects are the downstream nodes that populate the graph. Requires a Tracer object to add the correct proxies to the intervention graph.

Examples

The example below accesses the inputs and outputs of a gpt2 module, saves them during execution, and prints the resulting values:

with model.generate() as generator:
    with generator.invoke('The Eiffel Tower is in the city of') as invoker:
        hidden_states = model.lm_head.input.save()
        logits = model.lm_head.output.save()

print(hidden_states.value)
print(logits.value)

This example shows how to set the output of a gpt2 module to zero:

with model.generate() as generator:
    with generator.invoke('The Eiffel Tower is in the city of') as invoker:
        model.transformer.h[0].output[0] = 0
class nnsight.module.Module#

Class that wraps existing torch modules within a model’s module tree in order to add nnsight functionality. Proxies of it’s output and input are accessed by .output and .input respectively.

module_path#

String representing the attribute path of this module relative the the root model. Separated by ‘.’ e.x (‘transformer.h.0.mlp’). Set by NNsightModel on initialization of meta model.

Type:

str

output#

Proxy object representing the output of this module. Reset on pass through.

Type:

nnsight.intervention.InterventionProxy

output_shape#

Shape of the tensor outputs to this module. Populated by most recent pass through. Can also be a nested list of torch.Size.

Type:

torch.Size

output_type#

Dtype of the tensor outputs to this module. Populated by most recent pass through. Can also be a nested list of torch.dtype.

Type:

torch.dtype

input#

Proxy object representing the input of this module. Reset on pass through.

Type:

nnsight.intervention.InterventionProxy

input_shape#

Shape of the tensor inputs to this module. Populated by most recent pass through. Can also be a nested list of torch.Size.

Type:

torch.Size

input_type#

Dtype of the tensor inputs to this module. Populated by most recent pass through. Can also be a nested list of torch.dtype.

Type:

torch.dtype

tracer#

Object which adds this module’s output and input proxies to an intervention graph. Must be set on Module objects manually.

Type:

nnsight.context.Tracer.Tracer

property input: InterventionProxy#

Calling denotes the user wishes to get the input of this module and therefore we create a Proxy of that request. Only generates a proxy the first time it is references otherwise return the already set one.

Returns:

Input proxy.

Return type:

Proxy

property output: InterventionProxy#

Calling denotes the user wishes to get the output of this module and therefore we create a Proxy of that request. Only generates a proxy the first time it is references otherwise return the already set one.

Returns:

Output proxy.

Return type:

Proxy

static wrap(module: Module) → Module#

Wraps the torch Module with our Module class. Adds a forward hook to the module to populate output and input data. Does not wrap torch.nn.ModuleList.

Parameters:

module (torch.nn.Module) – The torch Module to wrap.

Returns:

The wrapped Module.

Return type:

Module
